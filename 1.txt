Rails로 상품 사진 업로드 기능 구현하기

이번 포스팅에서는 Rails 앱에서 상품 사진 업로드를 Active Storage로 구현한 과정을 정리합니다.

목표

상품에 사진을 업로드할 수 있도록 한다
상품 상세 페이지와 목록 페이지에 이미지를 보여준다
썸네일은 일정한 비율로 고정해서 UI를 깔끔하게 만든다
사용한 기술

Active Storage
image_processing gem
시스템 이미지 변환 라이브러리 libvips
Active Storage는 Rails 기본 기능이라 별도 서드파티 업로드 라이브러리 없이도 빠르게 구현할 수 있습니다.

설치 및 설정

Active Storage 설치
bin/rails active_storage:install
마이그레이션 실행
bin/rails db:migrate
Active Storage 마이그레이션을 생성
active_storage_blobs, active_storage_attachments 테이블이 추가됨
config/storage.yml에서는 기본적으로 local이 설정되어 있어 로컬 개발에서 바로 사용 가능합니다.

모델 변경

product.rb

class Product < ApplicationRecord
has_one_attached :photo
validates :name, presence: true
end

has_one_attached :photo를 추가하면 상품 모델에 이미지 첨부가 가능해집니다.
컨트롤러 변경

products_controller.rb

def product_params
params.require(:product).permit(:name, :photo)
end

업로드된 이미지 photo를 허용하도록 strong params에 추가했습니다.
업로드 폼 추가

new.html.erb 및 show.html.erb 수정

<%= form_with model: @product, local: true, html: { multipart: true } do |form| %>
...
<%= form.file_field :photo %>
...
<% end %>

multipart: true가 반드시 필요합니다
file_field :photo로 파일 입력을 추가합니다
이미지 표시

상세 페이지
show.html.erb

<% if @product.photo.attached? %>
<%= image_tag @product.photo.variant(resize_to_limit: [800, 600]) %>
<% end %>

목록 페이지
index.html.erb

<% if product.photo.attached? %>
<div class="aspect-[4/3] w-full overflow-hidden">
<%= image_tag product.photo.variant(resize_to_limit: [800, 600]), class: "h-full w-full object-cover" %>
</div>
<% end %>

상세 페이지는 원본 비율을 유지하면서 변환된 이미지를 노출
목록 페이지는 aspect-[4/3]로 고정 비율 썸네일 처리
썸네일 비율 고정

Tailwind의 aspect-[4/3]를 활용해서 썸네일 영역을 고정했습니다.

<div class="aspect-[4/3] w-full overflow-hidden"> <%= image_tag ... class: "h-full w-full object-cover" %> </div>

이미지가 카드 높이를 꽉 채우고
비율이 일정하게 유지되며
잘리는 부분은 깔끔하게 처리됩니다
추가로 필요한 시스템 라이브러리

이미지 변환을 위해 libvips가 필요합니다.

image_processing gem만 추가해도
시스템에 libvips가 없으면 variant 처리 실패
Ubuntu/Debian 환경 예:
sudo apt-get update
sudo apt-get install -y libvips libvips-dev

설치 후 Rails 서버를 다시 시작하면 정상 동작합니다.
